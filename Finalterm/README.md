## HW1 — P0 Compiler

A PL/0-style compiler that translates a Pascal-like subset language into bytecode executed on a **stack-based virtual machine**. Features include:

- Lexical scoping with static link resolution
- `while` loops with test-at-top semantics
- Lexically scoped procedure calls (supports call-by-value, recursion, and nested procedures)
- Bytecode generation for a custom stack-based VM

**Directory:** [HW1](../HW1/)

---

## HW2 — TinyLang Compiler

A **statically-typed, compiled programming language** designed for educational purposes and embedded systems. Implemented as a multi-pass compiler in C:

- **Lexer** — tokenizes source code into tokens
- **Parser** — recursive-descent parser with syntax validation
- **Code generator** — emits bytecode for the custom stack-based VM
- **VM** — portable stack-based virtual machine that executes the generated bytecode

**Directory:** [HW2](../HW2/)

---

## HW3 — Extended TinyLang Compiler

Enhances the HW2 TinyLang compiler with additional language features:

- **Arrays** with fixed-size allocation and bounds checking
- **For loops** with init/condition/increment clauses
- **Break & continue** for loop control flow
- **Logical operators** (`&&`, `||`, `!`)
- **Modulo operator** (`%`)
- **Else-if** chains
- **Comments** (`//` line, `/* */` block)
- Extended VM with array memory pool

**Directory:** [HW3](../HW3/)

## HW4 — System Programming Course Website

A **single-page interactive website** teaching system programming concepts (process management, memory, threads, file I/O, IPC, networking, assembly). Content sourced from Chinese-language course materials and translated to English.

Eight chapters covering: system calls, `fork()`/`exec()`, virtual memory, pthreads & mutex, file descriptors & pipes, shared memory & semaphores, TCP/UDP sockets, and ELF/assembly.

**Live Demo:** [system-programming-course.vercel.app](https://system-programming-course.vercel.app)
**Directory:** [HW4](../HW4/)

## HW5 — Concurrency and Synchronization

Three classic concurrency problems implemented in C using **POSIX threads (pthreads)**:

1. **Bank Deposit/Withdrawal** — multi-threaded bank account simulation demonstrating race conditions and mutex synchronization
2. **Producer-Consumer** — bounded-buffer problem with mutex and condition variables
3. **Dining Philosophers** — deadlock avoidance in resource allocation using mutex locks

**Directory:** [HW5](../HW5/)

---

## HW6 — Processes & File Descriptors

Three programs demonstrating **Unix process management and file descriptor manipulation** in C:

1. **file_copy** — copies files using `open()`/`read()`/`write()`/`close()` (unbuffered I/O)
2. **pipeline** — connects two processes with `pipe()` and `dup2()` for inter-process communication
3. **shell** — a minimal Unix shell supporting command execution, pipes, and I/O redirection

Key system calls: `fork`, `execvp`, `waitpid`, `pipe`, `dup2`, `open`, `read`, `write`, `close`.

**Directory:** [HW6](../HW6/)

## Midterm — Process Monitor

A **cross-platform terminal process monitor** (Linux, macOS, Windows) that displays real-time system resource usage and running process details:

- Live CPU, memory, swap, and system load statistics
- Process table with sortable columns (CPU time, memory, PID, name)
- Process filtering and kill functionality
- Color-coded output (red for high load, yellow for medium, green for normal)
- Keyboard-interactive interface with configurable refresh rate

**Directory:** [Midterm](../Midterm/)
